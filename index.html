Misc.

Glossary

  • ASLR: process memory at random addresses
  • CA: certificate authority; trusted verification of public key ownership
  • CRIME: compression ratio info-leak made easy; information leakage in compressed HTTPS responses
  • Cuckoo attack: convince user that an adversary-controlled TPM is in the user's own computer
  • Drawbridge: Windows containers using picoprocesses and library OS
  • ECID: Apple iPhone exclusive chip ID per device; used to sign iBoot and iOS
  • Guard model: gateway between separate systems (like firewall)
  • Iago attacks: attacks from malicious OS on an application
  • IMC: inter-module communications; NaCl uses NPAPI for this
  • MAC: mandatory access control
  • NPAPI: browser plugin framework used with NaCl; deprecated in favor of Pepper (PPAPI)
  • NX: non-executable (in the context of a stack)
  • PBKDF2: expensive hash function good for hashing passwords
  • RISC: reduced instruction set computer (e.g. MIPS)
  • SFI: software fault isolation
  • SGX: software guard extensions - enclave, attestation, sealing
  • stack canary: values store to protect stack from control flow hijacking
  • TCB: trusted computing base
  • TLS: transport layer security; private connection with symmetric cryptography
  • TPM: trusted platform module, secure cryptoprocessor that integrates cryptographic keys

Struct layout

Note that members are laid out in the order they are defined and each member is aligned to its own size. 
struct student 
{
    int id1;
    int id2;
    char a;
    char b;
    float percentage;
};

size of structure in bytes : 16

Address of id2 = 675376772
Address of a = 675376776
Address of b = 675376777
Address of percentage = 675376780
